# LLM Prompt Engineering Guide

## Overview

The LLM (Qwen3 32B via Ollama) is the "director" of our audiobook pipeline. It performs two critical tasks:
1. **Pass 1 — Character & Voice Analysis**: Read the book and create a character registry with voice descriptions
2. **Pass 2 — Script Generation**: Process each chapter into a line-by-line script with speaker attribution and emotion tags

This document contains the exact prompts, strategies, and examples used for each pass.

---

## Pass 1: Character & Voice Analysis

### Prompt Template

```markdown
You are an expert audiobook director preparing a novel for multi-voice narration. Your task is to analyze this book and create a character registry with voice descriptions for text-to-speech generation.

## Instructions

1. Read the following text carefully
2. Identify ALL speaking characters (anyone who has dialogue)
3. For each character, determine:
   - Their gender, approximate age, and key personality traits
   - A detailed voice description suitable for voice synthesis
4. Also create a narrator voice that fits the book's genre and tone
5. Output ONLY valid JSON — no explanation, no markdown code fences

## Voice Description Guidelines

Voice descriptions must be specific and actionable. Include:
- **Gender and age**: "young female, early 20s" or "elderly male, 70s"
- **Pitch**: "high-pitched", "deep baritone", "medium tenor"
- **Pace**: "fast-talking", "measured and deliberate", "slow and ponderous"
- **Quality**: "gravelly", "silky smooth", "raspy", "clear and bell-like"
- **Accent/Pronunciation**: "British RP", "no strong accent", "slight roughness"
- **Emotional baseline**: "warm and kind", "cold and calculating", "nervous energy"

Do NOT use real person names. Use archetypes instead.
Keep descriptions under 50 words each.

## Book Genre: Fantasy/Fiction

The narrator voice should suit epic fantasy storytelling — authoritative but warm, with gravitas for dramatic moments and warmth for intimate scenes.

## Output Schema

{
  "book_title": "string",
  "book_author": "string",
  "genre": "fantasy",
  "tone": "description of the book's overall tone",
  "characters": {
    "narrator": {
      "name": "Narrator",
      "gender": "male|female",
      "age_range": "string",
      "personality_traits": ["trait1", "trait2"],
      "voice_description": "detailed voice description for TTS",
      "speaking_style": "how the narrator typically speaks"
    },
    "character_id": {
      "name": "Character Display Name",
      "gender": "male|female|other",
      "age_range": "string",
      "personality_traits": ["trait1", "trait2"],
      "voice_description": "detailed voice description for TTS",
      "speaking_style": "how this character typically speaks"
    }
  }
}

## Book Text

{BOOK_TEXT}
```

### Strategy for Long Books

If the book exceeds the LLM's context window:
1. Send the first 3 chapters + last chapter (for character arc awareness)
2. Send a summary of each remaining chapter (generated by a separate LLM call)
3. The LLM may miss minor characters — these are caught in Pass 2 and assigned generic voices

### Handling Many Characters

Fantasy novels can have 30+ speaking characters. Strategy:
- **Major characters** (>10 lines of dialogue): Full voice description
- **Minor characters** (2-10 lines): Brief voice description based on gender/age
- **One-line characters** (1 line): Assigned a generic voice matching their gender
- Maximum unique voices: ~15-20 (beyond this, listeners can't distinguish anyway)

## Pass 2: Script Generation

### Prompt Template

```markdown
You are an expert audiobook script annotator. Convert the following chapter text into a structured audiobook script with speaker attribution, emotion tags, and pacing instructions.

## Context

### Character Registry
{CHARACTER_REGISTRY_JSON}

### Previous Chapter Summary (for emotional continuity)
{PREVIOUS_CHAPTER_SUMMARY}

## Instructions

1. Process the chapter text into individual speech segments
2. For each segment, determine:
   - **speaker**: Character ID from the registry, or "narrator" for non-dialogue
   - **text**: The exact text to speak (strip dialogue attribution like "he said")
   - **emotion**: Natural language description of the emotional state, considering context
   - **speed**: Delivery speed (0.8 = slow/dramatic, 1.0 = normal, 1.2 = fast/excited)
   - **pause_before_ms**: Silence before this segment (0-2000)
   - **pause_after_ms**: Silence after this segment (0-2000)

## Rules

### Speaker Attribution
- Anything inside quotation marks is dialogue — identify the speaker from context
- Everything outside quotation marks is narrator
- Internal monologue (often in italics or without quotes but clearly a character's thoughts) → assign to the thinking character with emotion "internal, thoughtful"
- If you can't determine the speaker, use "narrator"

### Emotion Tagging
- Consider the SURROUNDING CONTEXT, not just the words themselves
- "Fine" can be dismissive, relieved, or lying — the context tells you which
- Use 2-3 descriptive words: "angry, barely controlled" not just "angry"
- For narrator segments, the emotion reflects the SCENE MOOD, not a character's feelings
  - Action scene narration: "tense, urgent"
  - Peaceful description: "serene, contemplative"
  - Revelation: "ominous, building dread"

### Pacing
- Scene transitions: 1500-2000ms pause
- Paragraph breaks in narration: 500-800ms pause
- Between dialogue lines in conversation: 200-400ms pause
- Dramatic pauses within dialogue: 300-600ms pause_before
- After a dramatic revelation: 1000-1500ms pause_after
- Chapter opening: 1000ms pause_before on first segment
- Chapter ending: 2000ms pause_after on last segment

### Speed
- Normal narration: 1.0
- Action sequences: 1.05-1.1
- Dramatic/emotional moments: 0.85-0.9
- Whispered/secretive: 0.85
- Excited/panicked: 1.1-1.15
- Contemplative/sad: 0.85-0.9

### Text Cleaning
- Remove dialogue attribution ("he said", "she whispered", "Kvothe replied")
- Keep the actual spoken words only
- For narration, keep the full text
- Preserve special punctuation (ellipses, em-dashes) as they affect TTS pacing

### Segment Length
- Keep segments between 1-4 sentences
- Never split mid-sentence
- Split long narration paragraphs at natural breathing points
- Each dialogue utterance is one segment (even if it's one word)

## Output Schema

Output ONLY valid JSON:
{
  "chapter_number": int,
  "chapter_title": "string",
  "chapter_summary": "1-2 sentence summary for continuity with next chapter",
  "lines": [
    {
      "line_id": "ch01_001",
      "speaker": "character_id",
      "text": "The spoken text",
      "emotion": "descriptive emotion state",
      "speed": 1.0,
      "pause_before_ms": 0,
      "pause_after_ms": 500
    }
  ]
}

## Chapter Text

{CHAPTER_TEXT}
```

### Context Window Strategy

For each chapter, the LLM receives:
1. The character registry (from Pass 1)
2. The previous chapter's summary (for emotional continuity)
3. The full chapter text

If a chapter is too long for the context window:
1. Split into overlapping sections (~5000 words each, 500-word overlap)
2. Process each section separately
3. De-duplicate overlapping segments
4. Ensure speaker attribution is consistent across sections

### Handling New Characters

If the LLM encounters a speaking character not in the registry:
1. Assign them a temporary ID based on their description (e.g., "guard_1", "old_woman")
2. Generate a basic voice description from context clues
3. Add them to the registry with a flag `"discovered_in_pass2": true`
4. The voice bootstrapping stage will generate their reference clip

## Prompt Engineering Best Practices

### 1. Structured Output Enforcement
- Always end prompts with "Output ONLY valid JSON — no explanation, no markdown code fences"
- Include the exact schema in the prompt
- Validate LLM output against Pydantic models before proceeding

### 2. Emotion Specificity
**Bad**: `"emotion": "happy"`
**Good**: `"emotion": "joyful surprise, breathless with delight"`

The more specific the emotion description, the better Qwen3-TTS can modulate the voice.

### 3. Fantasy Name Handling
Fantasy books have unusual names (Kvothe, Auri, Ambrose). The LLM should:
- Use these names in voice descriptions ("this is the voice of Kvothe")
- Not try to phonetically spell them (the TTS handles pronunciation)
- Flag particularly unusual names that might trip up TTS

### 4. Temperature Settings
- Pass 1 (character analysis): Temperature 0.3 (we want consistent, factual output)
- Pass 2 (script generation): Temperature 0.4 (slightly more creative for emotion descriptions)
- Both: Top-p 0.9, no repeat penalty

### 5. Chunking Long Chapters
If a chapter has more than ~5000 words:
```
Chunk 1: words 0-5000 (full context)
Chunk 2: words 4500-9500 (500-word overlap with chunk 1)
Chunk 3: words 9000-13500 (500-word overlap with chunk 2)
...
```

After processing all chunks:
1. Remove duplicate segments from overlap regions
2. Verify speaker attribution consistency across chunk boundaries
3. Smooth emotion transitions at chunk boundaries

## Fallback Strategies

### LLM Returns Invalid JSON
1. Retry with same prompt (temperature variation may fix it)
2. If still failing, try wrapping output instruction: "Respond with ```json ... ```"
3. Extract JSON from markdown code fence
4. If all fails, use a simpler prompt that processes one paragraph at a time

### LLM Misidentifies Speaker
Common in:
- Scenes with 3+ characters speaking
- Indirect speech ("She told him that...")
- Dialogue without attribution

Mitigation:
- The 10-paragraph context window helps significantly
- Post-processing: check if the same speaker is attributed to consecutive contradictory lines
- Dashboard allows manual correction before TTS generation

### LLM Halluccinates Characters
Sometimes the LLM invents characters that don't exist. Mitigation:
- Cross-reference Pass 2 speaker IDs against Pass 1 character registry
- Unknown speakers get flagged (not rejected — they might be real minor characters)
- Dashboard shows a "new characters detected" alert

## Emotion Vocabulary Reference

For consistent emotion tagging, the LLM is guided to use these categories:

### Primary Emotions
| Category | Variants |
|----------|----------|
| **Joy** | delighted, ecstatic, content, amused, playful, warm, tender |
| **Sadness** | melancholic, grieving, wistful, lonely, resigned, heartbroken |
| **Anger** | furious, irritated, seething, cold fury, righteous indignation |
| **Fear** | terrified, anxious, nervous, dread, unease, paranoid |
| **Surprise** | shocked, astonished, bewildered, caught off guard |
| **Disgust** | revolted, contemptuous, disdainful, nauseated |

### Narrative Emotions (for narrator segments)
| Category | Variants |
|----------|----------|
| **Tension** | suspenseful, ominous, foreboding, urgent, taut |
| **Calm** | peaceful, serene, contemplative, meditative |
| **Energy** | dynamic, exciting, breathless, building momentum |
| **Mystery** | enigmatic, hinting, cryptic, conspiratorial |
| **Grandeur** | epic, sweeping, awe-inspiring, vast |
| **Intimacy** | quiet, personal, confessional, vulnerable |

### Modifiers
- Intensity: "barely contained", "explosive", "simmering", "gentle"
- Transition: "shifting from X to Y", "cracking into", "hardening into"
- Authenticity: "genuine", "forced", "masked", "performative"
